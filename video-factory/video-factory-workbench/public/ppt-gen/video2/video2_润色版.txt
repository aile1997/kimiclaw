# 《OpenClaw高阶进化指南》- 润色版（去除AI痕迹）

---

## 第1页：开场

先给你看个数据：12小时，13轮自动进化，写了8份文档，每次执行从128秒降到11秒。

不是秀操作，是我真的跑通了，而且这套方法你能直接用。

我是AI厂长，今天聊点硬核的——怎么让AI每天自己升级自己。

---

## 第2页：问题在哪

大部分人用AI，其实是**阻塞式交互**：
你发一句，AI回一段，你再看，再发。全程你得盯着。

我现在改成**异步架构**：
Cron定时触发，Subagents并行跑，跑完主动汇报。

单线程 vs 多线程，做过开发的都懂差距有多大。

---

## 第3页：架构怎么搭

三层结构：**调度层、执行层、沉淀层**。

底层Cron做调度，相当于给AI加了个`setInterval`，每小时触发一次。重点参数`sessionTarget: isolated`，任务跑在独立进程，不污染主会话。

中层Subagents，我配了`maxConcurrent: 8`，8个并发。每个子代理有自己的token预算和超时（300秒），一个崩了不影响别的。

顶层Skills，把经验写成可复用模块。其实就是把prompt工程化，封装成函数。

---

## 第4页：Cron配置

我的核心配置贴这儿：

```bash
openclaw cron add \
  --name "evolution_hourly" \
  --every 3600000 \
  --sessionTarget isolated \
  --thinking on \
  --message "执行进化任务..."
```

几个关键点：
- `every 3600000`：毫秒，3600000ms = 1小时
- `sessionTarget isolated`：隔离执行，出错不会抛到主会话
- `thinking on`：开深度思考，token多耗点，但质量稳

配完这行，AI跟后台服务一样，不用人管了。

## 第5页：Subagents并发

我同时跑了8个子代理，分工如下：

| 代理 | 任务 | 超时 |
|-----|------|------|
| searcher | 搜文档 | 120s |
| tester | 测功能 | 180s |
| writer | 写记录 | 60s |
| summarizer | 总结 | 90s |
| security | 扫风险 | 60s |
| analyzer | 分析数据 | 90s |
| web | 联网查 | 120s |
| archiver | 归档 | 60s |

每个代理独立跑，用`sessions_spawn`创建，生命周期完整：创建→执行→归档→清理。

并发的好处：总耗时 = 最慢的那个，而不是加起来。

## 第6页：Nested Subagents（2026.2.15新功能）

刚更新的功能，我测完了，思路清楚：

```
Main Agent（协调）
    └─ Orchestrator（分解任务）
           ├─ Worker A（子任务1）
           ├─ Worker B（子任务2）
           └─ Worker C（子任务3）
```

配置：
```json
{
  "subagents": {
    "maxSpawnDepth": 2,
    "maxChildrenPerAgent": 5,
    "maxConcurrent": 8
  }
}
```

适用场景：复杂任务要分解多层。比如主代理说"研究OpenClaw新功能"，Orchestrator拆成"搜索、测试、记录"，Worker各自执行。

注意：深度别设太高，上下文传递会损耗，我测下来2层刚刚好。

## 第7页：Skills工程化

13轮跑完，我攒了5个自定义Skill，本质是**Prompt as Code**。

拿`humanizer-zh`举例，它解决了AI写作的24种痕迹：
- 内容：过度强调意义、模糊归因、提纲式废话
- 语法：系动词回避、否定排比、三段式
- 风格：破折号太多、粗体滥用、表情符号
- 填充词："值得注意的是"、"必须指出的是"

安装：
```bash
npx skills add https://github.com/op7418/Humanizer-zh.git
```

调用：
```bash
/humanizer-zh 请人性化这段文字...
```

核心是可组合，像搭积木一样拼能力。

## 第9页：5个硬核技巧

**1. Cron延迟任务**
```bash
--at "+20m" --delete-after-run
```
20分钟后执行，跑完自动删，适合一次性提醒。

**2. 子代理超时**
```javascript
sessions_spawn({
  task: "分析数据",
  runTimeoutSeconds: 600,
  cleanup: "delete"
})
```
防止死循环，超时自动清。

**3. 思考级别切换**
```bash
/think minimal   # 简单问答，省token
/think high      # 复杂推理，保质量
```
成本差3-5倍，看情况选。

**4. Browser无障碍树**
```bash
browser snapshot --format aria --limit 50
```
用ARIA role定位，比CSS选择器稳，DOM变了也不怕。

**5. Memory向量搜索**
```bash
memory search "上次的决策" --max-results 5
```
基于embedding语义检索，不是关键词匹配，能找到相关但表述不同的内容。

## 第10页：成本与安全

**成本明细（13轮）**：
- Kimi API：~650k tokens × ¥0.002/1k = ¥1.3
- VectorEngine (Gemini中转)：~200k tokens × 免费 = ¥0
- 存储、网络：忽略不计
- **总计：¥1.8**（确实不到一杯奶茶）

**安全配置**：
```bash
# 1. Credentials目录权限
chmod 700 ~/.openclaw/credentials

# 2. 群组策略（建议从open改allowlist）
openclaw config set channels.feishu.groupPolicy allowlist

# 3. 插件白名单
openclaw config set plugins.allow ["feishu","web-search","browser"]
```

红线：子代理里别用`session`工具，除非`maxSpawnDepth >= 2`且你知道风险。

## 第11页：方法论

核心心法：**设计→编码→运行→沉淀的闭环**。

```
设计目标 → 编写配置 → 自动运行 → 经验沉淀
    ↑                                 ↓
    └──────── 持续迭代 ←──────────────┘
```

原则：**Text > Brain**

所有经验必须写成文件（Markdown/JSON/Skill），别指望脑子能记住。我现在`memory/`目录有完整日志，随时能回溯到任意一轮。

这是**可复现性**——科学实验的标准，也是AI工程化的基础。
